実プログラムとの整合性を自動検証済み： simulator/verify_against_c.mjs が Maze_Dijkstra.c を gcc -DSIM_EXPORT で実際にコンパイル・実行し、 その出力（各ノードのコスト・PASS/CP/NANAME 配列・最終位置と方向）をこのシミュレータの JS 移植と1項目ずつ突き合わせています。 デフォルト迷路でこの照合はすべて一致しました。壁を編集すると JS 側のみで再計算されるため、都度この検証コマンドが再実行されるわけではありません。
移植時に見つかった注意点： C側 (main() の経路復元ループ) では final_dir がループ開始時点の方向のまま更新されず、 ゴール到達直前の実際の向きではなく「スタート直後の向き」が使われています（last_dir が一度も再代入されない）。 これは実際の C プログラムの挙動そのもの（上記の検証で確認済み）なので、ロボットの初期姿勢設定に使っているなら仕様として問題ないか確認をおすすめします。
